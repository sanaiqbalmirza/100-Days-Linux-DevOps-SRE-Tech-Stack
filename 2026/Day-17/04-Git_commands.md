# IMPORTANT NOTE

When using git you will basically work 80% of the time using the following commands:

- Check Git repository status
- Stage files
- Commit changes
- Push code to GitHub
- Verify your uploaded files online

---

# FIVE STEPS ONLY!!

```bash
git status
git add .
git commit -m "README file updated"
git push
```

---

# Task 1 - Check Repository Status

## Objective

Check the current status of your Git repository.

---

## Run:

```bash
git status
```

---

## Observation

Check:

- Which files are modified
- Which files are untracked
- Which files are ready to commit

---

# Task 2 - Stage All Files

## Objective

Stage all files inside the repository.

---

## Run:

```bash
git add .
```

---

## Explanation

This command stages:

- README files
- Images
- Markdown documents
- Any modified files

for the next commit.

---

# Task 3 - Commit Your Changes

## Objective

Save your changes into Git history.

---

## Run:

```bash
git commit -m "README file updated"
```

---

## Explanation

- `commit` saves your work
- `-m` allows you to add a commit message

---

# Task 4 - Push Code to GitHub

## Objective

Upload your local changes to GitHub.

---

## Run:

```bash
git push
```

---

## Explanation

This command sends your committed files from:

```text
Local Repository
```

to:

```text
Remote Repository (GitHub)
```

---

# Task 5 - Verify Push on GitHub

## Objective

Confirm your files were uploaded successfully.

---

## Instructions

1. Open your GitHub Repository
2. Refresh the page
3. Verify that:
   - README files appear
   - Images appear
   - Markdown files appear
   - Latest commits are visible

---

## IMPORTANT NOTE

Do NOT forget to refresh your GitHub page after pushing.
